# 💳 Data-Driven Credit Card Fraud Detection

A complete machine learning pipeline for detecting fraudulent credit card transactions. This application covers data preprocessing, exploratory data analysis (EDA), model training with improvement techniques, and a professional web interface for real-time fraud risk assessment.

## 📊 Dataset Overview

The project uses two CSV datasets (`Credit_Card_Dataset_2025_Sept_1.csv` & `Credit_Card_Dataset_2025_Sept_2.csv`) containing:
- **25,134 transactions** across **19 features** (demographic, financial, and contact data)
- **Target variable:** Binary (0 = Legitimate, 1 = Fraud)
- **Class imbalance:** Only 422 fraud cases (1.7%) — addressed using SMOTE and threshold optimisation

## 🚀 Getting Started

### Prerequisites (Ubuntu / Linux)
- Python 3.8 or higher
- Install Python and pip if not already installed:

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv
```

- (Optional) Create and activate a virtual environment:

```bash
python3 -m venv venv
source venv/bin/activate
```

- Install project dependencies:

```bash
pip3 install -r requirements.txt
```

### Running the Application

1. **Train the models:** Runs the full pipeline (merge → preprocess → EDA → model training) and saves outputs.
   ```bash
   python3 train.py
   ```

2. **Launch the web interface:** Starts the Flask dashboard for data exploration and real-time predictions.
   ```bash
   python3 run.py
   ```
   Then open `http://127.0.0.1:5000` in your browser.

## 🤖 Machine Learning Models

Two models were built and compared:

| Model | Accuracy | Precision | Recall | F1-Score | ROC-AUC |
| :--- | :---: | :---: | :---: | :---: | :---: |
| Random Forest (Baseline) | 98.1% | 0.41 | 0.26 | 0.32 | 0.76 |
| Logistic Regression | 62.1% | 0.03 | 0.69 | 0.06 | 0.70 |
| **RF + Tuned Threshold** | **97.6%** | **0.32** | **0.42** | **0.36** | **0.80** |

### Improvements Applied
1. **GridSearchCV:** Systematic hyperparameter tuning across 36 parameter combinations
2. **Threshold Optimisation:** Decision boundary moved from 0.5 → 0.30 to maximise F1-score, boosting recall by 59%
